# Skill — outcome-roadmap

## Objetivo
Montar roadmap orientado a outcomes e OKRs, não a features ou datas fixas.

## Input
```json
{ "okrs": [], "horizonte": "trimestre | semestre | ano" }
```

## Output
```json
{
  "roadmap": [
    {
      "periodo": "Q1 2025",
      "objetivo": "...",
      "krs_alvo": [],
      "oportunidades_a_atacar": [],
      "hipoteses_iniciais": []
    }
  ]
}
```

## Passos
1. Organizar OKRs por período
2. Associar oportunidades priorizadas a cada período
3. Evitar comprometer features ou soluções específicas
4. Sinalizar dependências entre períodos
