# Skill — continuous-interview-partner

## Objetivo
Estruturar roteiros de entrevista contínua (cadência semanal, conforme Torres)
e sintetizar os principais insights por sessão.

## Input
```json
{ "oportunidade_alvo": "...", "perfil_do_entrevistado": "...", "suposicoes_a_testar": [] }
```

## Output
```json
{
  "roteiro": { "abertura": "...", "perguntas": [], "encerramento": "..." },
  "insights_por_sessao": [],
  "padroes_identificados": [],
  "novas_oportunidades_descobertas": []
}
```

## Regra (Torres)
Perguntas devem explorar comportamentos passados reais, nunca opiniões sobre
funcionalidades hipotéticas. Ex: "Me conta a última vez que você..." não "Você usaria...?"
