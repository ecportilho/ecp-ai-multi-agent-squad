# Skill — adr-writer

## Objetivo
Registrar Architecture Decision Records com contexto, decisão e tradeoffs.

## Formato ADR
```markdown
# ADR-001 — [Título]

## Status
Aceito | Proposto | Substituído

## Contexto
[O que motivou esta decisão]

## Decisão
[A decisão tomada]

## Consequências
**Positivas:** ...
**Negativas:** ...
**Neutras:** ...

## Alternativas Consideradas
- [Alternativa A] — descartada porque ...
- [Alternativa B] — descartada porque ...
```

## ADRs Base do o produto
- ADR-001: Persistência em JSON ao invés de banco de dados relacional
- ADR-002: Monolito modular ao invés de microserviços
- ADR-003: Front End HTML/CSS/JS puro sem framework
